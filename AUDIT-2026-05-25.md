## Critical

### C1. Chat WebSocket fails with HTTP 403 (chat is non-functional)
- **Console:** `WebSocket connection to 'wss://book-club-be.onrender.com/chat/rooms/47242bd9-18d4-4595-a44e-54f0be0a3dc5?token=...' failed: Error during WebSocket handshake: Unexpected response code: 403`
- **Repro:** Login → open any club → click "Відкрити чат". The room loads message history via REST (`GET /api/v1/chat/rooms/<id>/messages` → 200), but the WS handshake is rejected.
- **Impact:** No live message delivery; the "Надіслати" button stays disabled even after typing. Optimistic local rendering shows the typed message but it is never broadcast — confusing.
- **Possible cause:** backend WS auth checks token differently than REST, or membership check fails because room belongs to a club the user just joined but membership cache is stale.

### C2. `/manage` route is a dead placeholder
- **Repro:** As organizer, navigate to `/manage`. Renders the same `ClubsListComponent` as `/clubs` with no organizer-specific actions (no Create/Edit/Delete club, no Manage members).
- **Impact:** Organizer role is essentially read-only in the UI. There is also no link in the header navigation to reach this route.

## High

### H1. Joining an event silently auto-joins the parent club
- **Repro:** Logged in as a brand-new user with 0 clubs joined. On `/events`, click "Участь" on an event ("Травнева оргія" / Super Puper club). No confirmation prompt. Open `/clubs` → tab "Мої клуби (1)" now lists Super Puper as "✓ Учасник".
- **Impact:** Membership side-effect is not disclosed. Quitting the event does not necessarily remove the club membership (not verified, but worth confirming).

### H2. "Мої події" tab is empty although the user attends an event
- **Repro:** On `/events`, attend "Травнева оргія", then switch to "Мої події" tab → "Немає майбутніх подій. Приєднуйтесь до клубів, щоб бачити їхні події тут."
- **Impact:** Either the filter logic is wrong (should include events I'm RSVP'd to) or the helper text is wrong (it talks about clubs, not RSVPs). Both interpretations are user-visible bugs.

### H3. Profile statistics out of sync with reality
- **Repro:** After joining Super Puper (visible in "Мої клуби 1"), open `/profile` → "Клубів приєднано: 0".
- **Impact:** Stats counters are not refreshed (either backend not incremented, or frontend cache not invalidated). Likely affects "Квізів пройдено / виграно", "Книг прочитано" too.

### H4. No "Create club" / "Create event" UI for the Organizer role
- **Repro:** Toggle role to Organizer in `/profile` → "✓ Активний" updates. Navigate to `/clubs` and `/events`. There is no Create button on either page, and no "Manage" link in the header.
- **Impact:** Organizer features (the only differentiator vs. Reader) are not reachable from the UI.

### H5. Chat rooms list contains many duplicate/leaked rooms
- **Repro:** `/chats` → sidebar shows in Super Puper club: 10× rooms named "igor", plus "dd", "fd", "ikzgf", "Травнева оргія · Chat", etc.
- **Impact:** No de-duplication, no minimum length validation, no archive. Production data is polluted by test rooms.

## Medium

### M1. Inconsistent localization of the RSVP button
- "Участь" (events list, not yet attending)
- "✓ Іду" (events list, already attending)
- "✓ Йду" (club detail card) — different verb form
- "✓ Іду · Скасувати" (event detail page) — adds dot-separated cancel hint
- **Impact:** Same action, three different labels.

### M2. "На головну" link on /login and /register routes to a page that doesn't exist for guests
- **Repro:** As unauthed user open `/login` → click "← На головну". URL goes to `/`, which the auth guard immediately redirects back to `/login`. The link is effectively a no-op (or a refresh).
- **Impact:** Misleading affordance — there is no public landing page.

### M3. Back link on `/privacy` and `/terms` always goes to `/clubs`
- **Repro:** Open `/privacy` while logged out → click "← Назад" → lands on `/login` after redirect. While logged in, lands on `/clubs` even if you arrived from `/events`.
- **Impact:** Should use `history.back()` or context-aware target.

### M4. Two "Відкрити чат" buttons visible at the same time on the club detail page
- **Repro:** Open any club detail. There is the inline "Відкрити чат" inside the "Про клуб" card AND the floating chat FAB at the bottom-right.
- **Impact:** Visual clutter, unclear which one to use. The floating FAB also remains on top of the chat panel after the panel opens.

### M5. Send button stays disabled even when text is present
- **Repro:** In the chat panel, type into "Написати повідомлення..." — the "Надіслати" button remains `[disabled]`. Pressing it via DOM still produces an optimistic UI entry, but the disabled state is wrong.
- **Impact:** Users see the button as unusable; combined with C1 this is doubly broken.

### M6. Duplicate "Закрити чат" buttons in chat panel header
- The chat panel header renders two buttons with the same accessible name "Закрити чат" (one is the X icon, the other appears outside the panel). Hurts a11y.
